# 0.2. Конвенции для всех страниц (единый шаблон)

- **Last reviewed**: 2026-01-15
- **Уровень**: Basic

## Зачем эта страница

Чтобы страницы читались как методичка, каждая “wiki‑страница” следует одному формату: **метаданные → зачем → как применять → чек‑лист → шаблоны → риски → ссылки**.

## Верхний блок (метаданные)

В начале каждой страницы:

- **Last reviewed**: дата
- **Уровень**: Basic / Advanced

## Структура контента (скелет)

Рекомендуемые секции (если применимо):

1. **Зачем эта страница**
2. **Когда использовать / когда НЕ использовать**
3. **Ключевые понятия**
4. **Пошагово** (процесс/чек‑лист)
5. **Шаблоны** (prompts / инструкции / диаграммы Mermaid)
6. **Риски и анти‑паттерны**
7. **Связанные страницы**
8. **Внешние ссылки**

## Термины, которые используем консистентно

- **Spec / AC / DoD**: что хотим получить и как проверяем.
- **Plan**: список шагов + список файлов + риски + тест‑план.
- **Context Pack**: минимально‑достаточный пакет контекста, без “мусора” и без секретов.
- **STOP‑POINT**: контрольная точка, где агент обязан остановиться и запросить подтверждение.
- **Human‑in‑the‑loop**: обязательная обратная связь на многошаговых изменениях (особенно A2+).
