# Cursor rules (template)

## Идея

Cursor Rules — это “постоянные” правила проекта, которые действуют шире, чем разовые команды.

## Рекомендуемая структура (пример)

- `.cursor/rules/`
  - `global.md` — общие правила проекта (DoD, safety, формат ответа)
  - `backend.md` — правила для backend (архитектура, тесты, миграции)
  - `frontend.md` — правила для frontend (стиль, компоненты, тесты)
  - `security.md` — зоны повышенного риска и запреты

## Что включать (чек‑лист)

- “Plan first” для мультифайл задач
- STOP‑POINT перед запуском команд
- запрет секретов/PII в контексте
- “не трогать” зоны (auth/crypto/CI/db migrations)
- предпочтительный формат результата (план → патч → проверки)
