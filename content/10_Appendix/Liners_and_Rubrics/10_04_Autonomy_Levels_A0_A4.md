## Зачем эта страница

Разрешить агентность там, где это безопасно, и запретить там, где дорого.

## Уровни (A0–A4)

### A0 — Только советы (No write)

- агент не меняет код, только объясняет/предлагает
- примеры: разбор модуля, предложение плана

### A1 — Локальные правки (1 файл) + diff

- можно менять один файл и показывать patch
- примеры: переименования, маленькие условные правки

### A2 — Мультифайл в пределах фичи + план

- требования: сначала план + список файлов, затем изменения
- запрет “ходить по всему репо”
- примеры: endpoint + сервис + тест

### A3 — Запуск проверок по подтверждению

- агент может **предложить** команды (tests/lint/build), но запуск — только после явного approve

### A4 — Интеграционные действия запрещены по умолчанию

- push/merge/deploy делает только человек
- изменения CI/secrets/auth — только с ревью и по процессу

## Политика по умолчанию

Всё выше A2 → только для подготовленных репо/команд и при наличии guardrails (см. [6.5](../../06_Security_Compliance/06_05_Advanced_Guardrails.md)).
