# 10.4.U. Сегменты пользователей (U1–U5)

- **Last reviewed**: 2026-01-15
- **Уровень**: Basic

## Зачем эта страница

Понять, что людям нужно по‑разному: кому-то “5 золотых промптов”, кому-то — guardrails и локальный контур.

## Оси сегментации

1. **Частота**: редко / еженедельно / ежедневно
2. **Режим**: autocomplete / чат / агент (end‑to‑end)
3. **Тип задач**: рутина / фичи / дебаг / рефакторинг / онбординг
4. **Риск‑профиль**: “сделай быстро” vs “сделай безопасно”
5. **Контекст‑дисциплина**: фрагменты / файлы / папки / AC+логи+данные

## Сегменты

- **U1. Explorers (Пробуют)**  
  Признаки: 1–2 раза в неделю, больше чат, мало контекста.  
  Что дать: 5–7 “золотых” промптов + примеры Context Pack + короткий гайд “как проверять”.

- **U2. Routine Accelerators (Ускорители рутины)**  
  Признаки: ежедневно, много boilerplate, часто autocomplete/короткий агент.  
  Что дать: библиотека шаблонов, правило “минимальный diff”, быстрый чек‑лист для PR.

- **U3. Agentic Builders (Агенты на фичи/рефактор)**  
  Признаки: agent end‑to‑end, много файлов, выше риск регрессий.  
  Что дать: уровни автономности + guardrails, политика “план → diff → тесты”, лимиты на массовые правки.

- **U4. Risk‑Averse Experts (Осторожные профи)**  
  Признаки: доверие умеренное, проверяют всё, используют для черновика/разбора.  
  Что дать: “объясни/докажи/покажи инварианты”, шаблоны тест‑стратегии, доказательный prompting.

- **U5. Local‑first / Sensitive Data (Локальные/данные)**  
  Признаки: выбирают on‑prem, боятся утечек, нужен RAG по внутренним данным.  
  Что дать: политика маршрутизации данных + quickstart локальных моделей + RAG/MCP playbook.
