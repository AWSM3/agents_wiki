## Зачем эта страница

Инструкции — это “операционная система” агента. Без единого стандарта каждый пишет по‑своему и результаты не повторяются.

## Какие типы инструкций бывают

- **AGENTS.md**: универсальный “README для агента” (постоянный контекст проекта).
- **IDE Rules** (например, Cursor rules): правила, действующие на проект/папку.
- **Commands / workflows**: интерактивные многошаговые процедуры (slash‑команды).
- **Tool‑specific instructions**:
  - Copilot instructions
  - CLAUDE.md для Claude Code

## Мэппинг (рекомендация)

Источник истины — AGENTS.md, а tool‑специфичные файлы:

- повторяют ключевые guardrails
- добавляют “как именно” в конкретном инструменте (пути, настройки)

## “Правила без утечек”

Как описывать домен безопасно:

- без реальных доменных сущностей/таблиц/индексов
- через абстракции и контракты (AC/инварианты)
- через “золотые примеры” на синтетике
