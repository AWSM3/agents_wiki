# 5.5. Playbook: дебаг по логам/трассам/профилям

- **Owner**: SRE / Dev (TBD)
- **Статус**: Draft
- **Last reviewed**: 2026-01-15
- **Уровень**: Basic → Advanced
- **Для ролей**: Dev / SRE

## Когда применять

- есть ошибка в логах/трейсах/профилях
- нужен быстрый список гипотез и минимальный фикс

## Input

- обезличенный лог/трейс (redaction обязателен)
- путь до релевантной функции/модуля
- условия воспроизведения (repro)

## Output

- гипотезы (2–5)
- эксперименты/проверки
- минимальный фикс + тест (если уместно)

## Process

1. Redaction: скрыть PII/секреты/внутренние hostnames.
2. Попросить: гипотезы → проверки → минимальный фикс.
3. Verify (тест/локальный прогон).

## Safety

- запрещено делиться prod‑логами с PII/секретами во внешний контур

## Template prompt

> Вот обезличенный лог/trace: …  
> Repro: …  
> Где искать: файл/функция …  
> Дай 3–5 гипотез, затем минимальный фикс и как проверить.
