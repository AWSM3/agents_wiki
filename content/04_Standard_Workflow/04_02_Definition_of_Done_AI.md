# 4.2. Definition of Done (DoD) для AI‑изменений

- **Owner**: Engineering Managers / DevEx (TBD)
- **Статус**: Draft
- **Last reviewed**: 2026-01-15
- **Уровень**: Basic
- **Для ролей**: Dev / TL / QA / SRE

## Зачем эта страница

Чтобы AI‑ассистированные изменения были проверяемыми и безопасными: “сделано” означает “можно мержить”.

## DoD (минимальный)

- Код **собирается/компилируется** локально и в CI (если CI есть).
- Тесты **добавлены/обновлены** (или явно обосновано, почему не нужно).
- Пройдены статические проверки (линтер/форматтер), если применимо.
- Пройден secret scan / нет утечек секретов (или ручная проверка, если сканера нет).
- Нет “магических” изменений:
  - агент/автор может объяснить каждую правку “что и зачем”
  - нет лишних изменений вне области задачи (минимальный diff)
- Не расширен attack surface без согласования (особенно: auth/crypto/CI/DB/network).
- Документация/ADR обновлены, если было архитектурное изменение.
- В PR/MR есть:
  - ссылка на Spec/AC
  - “как проверить”
  - риски и rollback

## Усиленный DoD (рекомендуется для A2+ / S2+)

- Добавлен тест на регрессию (если фиксит баг).
- Прогнан полный набор проверок (по проекту).
- Ревью зон повышенного риска (CODEOWNERS: security/db/auth).
