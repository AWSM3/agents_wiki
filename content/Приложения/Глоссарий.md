# 10.1. Глоссарий (LLM, агент, MCP, RAG, tool‑use…)

- **Last reviewed**: 2026-01-15
- **Уровень**: Basic

## Core

- **AI (Artificial Intelligence)** — искусственный интеллект; в контексте — ассистент, помогающий писать/анализировать код и тексты.
- **IDE (Integrated Development Environment)** — среда разработки (например, Cursor/VS Code/IntelliJ) с редактором, запуском тестов, отладкой и т.п.
- **PR (Pull Request), MR (Merge Request)** — запрос на вливание изменений в основную ветку (код‑ревью и проверка перед merge).
- **DoD (Definition of Done)** — «определение готовности»: чек‑лист условий, при которых работа считается завершённой.
- **AC (Acceptance Criteria)** — критерии приёмки: конкретные проверяемые условия “что должно работать”, чтобы задача считалась выполненной.
- **ROI (Return on Investment)** — эффект от AI (время/качество) относительно затрат (время, деньги, риски).

## Метрики, аналитика, оценка

- **Baseline** — базовая линия сравнения (“как было без AI”): время, итерации ревью, дефекты и т.п.
- **Scorecard** — карточка/форма оценки кейса (структурирует измерения “до/после”, качество, вывод).
- **One-pager** — документ “на одну страницу” (сжатое резюме эффектов/рисков/плана).
- **WAU (Weekly Active Users)** — недельная активная аудитория: доля людей, использовавших AI ≥ 1 раз за неделю.
- **Rubric** — рубрика/шкала оценивания (набор критериев “как оценивать качество/готовность”).

## Разработка, репозиторий, проверки

- **Repo / Repository** — репозиторий с кодом (Git).
- **Diff** — разница между версиями файлов; “минимальный diff” = изменения только по сути задачи.
- **Refactoring (Рефакторинг)** — изменение структуры кода без изменения внешнего поведения.
- **Regression (Регрессия)** — “сломалось то, что раньше работало” из‑за новых изменений.
- **Linter / Линтер** — инструмент статического анализа, проверяющий стиль/ошибки/правила.
- **Unit test / Integration test** — юнит‑тест / интеграционный тест.
- **Repro (Reproduction steps)** — шаги воспроизведения бага.

## CI/CD и Git‑операции

- **CI (Continuous Integration)** — автоматические сборки/тесты/проверки при изменениях.
- **CD (Continuous Delivery/Deployment)** — автоматизация выката.
- **Protected branch** — защищённая ветка (обязательные ревью/проверки).
- **Pre-commit** — хук/проверки перед созданием коммита (форматирование, линт, секреты).

## Безопасность и управление рисками

- **Guardrails** — “ограждения/ограничения”, снижающие риск: технические и процессные правила для AI и кода.
- **Hard guardrails** — жёсткие технические ограничения (protected branches, policy checks, секрет‑сканеры).
- **Soft guardrails** — процессные ограничения (правила промптинга, чек‑листы, обязательный план).
- **Secret scanning** — автоматический поиск секретов (ключей/токенов) в коде/коммитах/PR.

## AI/LLM, режимы, агентность

- **LLM (Large Language Model)** — большая языковая модель.
- **Prompt / Промпт** — текст инструкции модели.
- **Reasoning mode** — режим “усиленного рассуждения” (обычно медленнее/дороже, но надёжнее).
- **Autocomplete** — автодополнение кода в редакторе.
- **Agent / Агент** — ассистент, способный на автономность: **планирует шаги и выполняет действия** (файлы, патчи, команды).
- **LLM-as-judge** — использование LLM для проверки/оценки результата.

## Контекст и артефакты знаний

- **Context / Контекст** — данные, которые вы даёте модели: фрагменты кода, файлы, логи, требования.
- **Context Pack** — стандартизированный набор контекста для задачи: цель, ограничения, файлы, AC, логи/payload.
- **Spec-driven / SDD** — подход “от спецификации”: сначала требования/AC, затем реализация и проверки.
- **Playbook** — согласованные правила/шаблоны/лучшие практики (как делаем X).

## Данные, локальные модели, интеграции

- **On-prem** — развёрнуто “внутри контура” компании.
- **Local-first** — принцип: по умолчанию использовать локальные/внутренние ресурсы для чувствительных задач.
- **Gateway** — шлюз/прокси к моделям/инструментам (точка контроля доступа/логирования/политик).
- **RAG** — генерация с “подтягиванием” фактов из источников (доки/репо/Confluence).
- **Indexing (Индексация)** — подготовка источников для RAG.
- **MCP** — протокол подключения внешних инструментов и источников данных к ассистенту.

## “Линейки” (коротко)

- **E0–E4** — уровни зрелости AI‑инжиниринга (см. [10.4](Liners_and_Rubrics/10_04_Liners_Index.md)).
- **A0–A4** — уровни автономности агента.
- **S0–S3** — серьёзность фейла/дефекта.
- **D0–D3** — классы чувствительности данных.
- **L1–L3** — уровни владения навыками.
- **U1–U5** — сегменты пользователей по паттернам применения AI.
