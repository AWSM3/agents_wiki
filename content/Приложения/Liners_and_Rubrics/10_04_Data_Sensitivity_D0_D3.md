# 10.4.D. Чувствительные данные и локальные модели (D0–D3)

- **Last reviewed**: 2026-01-15
- **Уровень**: Basic

## Зачем эта страница

Снять страх “утечек” и дать простое правило маршрутизации: **что куда можно отправлять**.

## Классификация данных (пример)

- **D0 Public**: можно в облако
- **D1 Filtration**: можно в облако с ограничениями (без секретов)
- **D2 Sensitive**: только on‑prem/локально
- **D3 Restricted**: нельзя в LLM вообще (или только строго изолированный контур)

## “Что куда” (примеры)

- prod‑логи с персональными данными → D2/D3
- конфиги, ключи, токены → D3
- фрагменты бизнес‑логики без данных клиентов → D1/D2 (по локальной политике)
- доки/гайдлайны без секретов → D1
