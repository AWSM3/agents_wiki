# 10.4.S. Таксономия фэйлов и матрица серьёзности (S0–S3)

- **Last reviewed**: 2026-01-15
- **Уровень**: Basic

## Зачем эта страница

Превратить "галлюцинации" в управляемые категории, чтобы улучшать процесс и снижать регрессии.

## Таксономия (пример)

1. **Logic Wrong** — логика неверна, баг не исправлен
2. **Regression Risk** — ломает существующее поведение
3. **Architecture Mismatch** — не следует паттернам проекта
4. **Overengineering** — слишком сложно, лишние абстракции
5. **Context Miss** — игнорирует контекст репо/ограничения
6. **Unsafe Fix** — "чинит" удалением логики/логов/проверок
7. **Test Gap** — тесты фиктивные/не ловят регресс
8. **Tooling/Format** — команды/формат ломают пайплайн

## Шкала серьёзности (S0–S3)

- **S0**: косметика/стиль
- **S1**: локальная ошибка, легко заметить, исправление < 30 минут
- **S2**: ошибка логики/регрессия, заметно позже, исправление часы
- **S3**: security/perf/инцидентный риск, потенциально критично

**Правило**: всё S2–S3 → обязательно "план → diff → тесты" + ручной ревью.
