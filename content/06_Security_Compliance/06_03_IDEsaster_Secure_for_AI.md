# 6.3. IDEsaster и “Secure for AI” (практические выводы для политики)

- **Last reviewed**: 2026-01-15
- **Уровень**: Advanced

## Зачем эта страница

Зафиксировать правила на случай, когда “IDE‑агент” становится вектором атак: untrusted контент может пытаться заставить агента выполнить вредные действия.

## Практические выводы (policy‑level)

- **Запрещаем auto‑run** команд по умолчанию.
- **Не даём агенту “кард‑бланш”**: доступ к терминалу/сети/секретам должен быть ограничен.
- **Untrusted файлы/логи/тикеты** рассматриваем как потенциальную prompt‑injection поверхность.
- **Минимизируем доступ к секретным папкам и конфигам** (особенно `.env`, kubeconfigs, CI secrets).
- **Явные STOP‑POINT’ы** на массовые правки и интеграционные действия.

## Чек‑лист для команд

- есть правила “что нельзя трогать” (auth/db/crypto/CI)
- есть уровни автономности и ограничения tool‑use
- есть секрет‑сканирование и review зон риска
