# 6.6. Настройки privacy/retention по инструментам (практический справочник)

- **Last reviewed**: 2026-01-15
- **Уровень**: Basic → Advanced

## Зачем эта страница

Чтобы у команды было единое место, где описано: **какие настройки нужно включить**, и **где проверить**, что данные не удерживаются/не используются не по политике.

## Как использовать

1. Для каждого “approved” инструмента заведите **Tool Card** (см. 3.3).
2. В Tool Card зафиксируйте:
   - режим privacy (если есть)
   - политику retention/логирования
   - настройки индексации/контекста (что исключаем)
   - кто владелец и когда пересматриваем
3. Пересматривайте раз в квартал (см. [9.3](../09_Governance_Adoption/09_03_Freshness_Policy.md)).

## Мини‑чек‑лист (для любого инструмента)

- есть ли режим “privacy” / “no training”
- сохраняются ли prompts/outputs, как долго, где
- кто имеет доступ к логам/трассам/истории
- какие интеграции включены (MCP, плагины, web access)
- исключены ли секретные папки из индексации/контекста

## Пример: инструменты, которые обычно встречаются

> Подробности всегда сверяйте с официальными документами и договором вашей организации.

- **Cursor**: privacy mode + правила `.cursor/rules`, контроль команд/auto‑run.
- **Claude Code**: проектный `CLAUDE.md`, ограничения tool‑use, хуки/плагины (если используете).
- **GitHub Copilot**: repo instructions + политика удержания по плану (Business/Enterprise).
- **OpenRouter/шлюзы**: режимы ZDR и маршрутизация к провайдерам по политике.
