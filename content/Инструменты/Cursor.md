# Tool Card: Cursor (IDE‑агент)

- **Last reviewed**: 2026-01-15
- **Уровень**: Basic → Advanced

## Что это

IDE‑агент, который умеет работать с кодовой базой: находить файлы, предлагать планы и вносить мультифайл‑изменения с патчами.

## Где запускается

- локально в IDE (рабочее место разработчика)

## Ключевые возможности

- чат + агентные правки по проекту
- планирование (Plan‑подход для сложных задач)
- работа с "правилами" проекта (rules) и командами (commands)
- запуск команд (по политике подтверждений)

## Как задавать правила/инструкции

- **Rules**: `.cursor/rules/*` (версионируются, можно делать scoped по путям)
- **Commands**: `.cursor/commands/*.md` (slash‑команды, многошаговые workflow)
- **AGENTS.md**: как универсальный source‑of‑truth (см. [10.3](../Приложения/Шаблоны/Главная.md))

## Privacy/retention (что проверить)

- включён ли privacy‑режим (если поддерживается вашим планом)
- какие папки исключены из индексации/контекста (секреты, prod‑конфиги)
- запрет auto‑run команд по умолчанию

## Риски/ограничения

- prompt injection через файлы/README/логи
- "ползучий scope": агент меняет больше, чем нужно
- риск утечек, если в контекст попадает D2/D3

## Мини‑рецепты

1) **Мультифайл фича (A2)**  
Попросить: план → список файлов → риски → тест‑план → (OK) → патч → проверки.

2) **Дебаг по логам**  
Дать обезличенный лог + указать файл/функцию + запретить "сеть/секреты" → попросить гипотезы и минимальный фикс + тест.

3) **PR‑ревью**  
Попросить отчёт по рубрике: correctness/security/perf/style/tests + severity.
