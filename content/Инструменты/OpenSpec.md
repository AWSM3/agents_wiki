# Tool Card: OpenSpec (spec‑driven workflow для ассистентов)

- **Last reviewed**: 2026-01-15
- **Уровень**: Advanced

## Что это

Подход/инструментарий для spec‑driven разработки с ассистентами: фиксируем спецификацию и артефакты, затем генерируем/вносим изменения по процессу.

## Где запускается

- обычно в репозитории (markdown артефакты + команды/интеграции)

## Как использовать

- ввести единый стандарт артефактов (Spec/ADR/TestPlan)
- связать с workflow 4.1
- ограничить автономность и добавить STOP‑POINT’ы

## Риски

- без культуры “verify” превращается в “vibe coding”
- требует владельцев и процесса актуальности
