# Tool Card: Claude Code (terminal‑агент)

- **Last reviewed**: 2026-01-15
- **Уровень**: Basic → Advanced

## Что это

Terminal‑агент для многошаговых задач в репозитории: анализ, правки, подготовка патчей, (по подтверждению) запуск команд.

## Где запускается

- терминал (локально или на выделенной машине)

## Ключевые возможности

- удобно вести длинные “процедуры” (миграции, CI, скрипты)
- можно использовать `CLAUDE.md` как постоянный контекст проекта
- подходит для plan‑then‑act и report‑style результатов

## Как задавать правила/инструкции

- `CLAUDE.md` в корне репозитория (persistent context)
- `AGENTS.md` как общий source‑of‑truth (см. [10.3](../Приложения/10_03_Templates_Index.md))
- правила на запуск команд: только после approve

## Privacy/retention (что проверить)

- где хранится история/логи сессий
- как инструмент обрабатывает локальные файлы/индексацию
- политика сети и интеграций

## Риски/ограничения

- терминал = высокий риск: эксфильтрация/вредные команды без контроля
- prompt injection через файлы/логи

## Мини‑рецепты

1) **Рефакторинг с проверками (A2–A3)**  
План → патч → (approve) → запуск тестов → исправить падения.

2) **Анализ репозитория**  
Попросить карту модулей, точки входа, риски, предложения по улучшениям без правок.
