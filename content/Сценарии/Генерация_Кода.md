# 5.1. Playbook: генерация кода/скелетов

- **Last reviewed**: 2026-01-15
- **Уровень**: Basic

## Когда применять

- новый модуль/endpoint/worker
- нужно быстро получить "walking skeleton"

## Input (что даём агенту)

- Spec/AC (или минимум: цель + ограничения)
- архитектурные ограничения (структура проекта, стиль)
- запреты по данным/секретам

## Output (что ожидаем)

- запускаемый skeleton (минимальный вертикальный срез)
- базовые тесты/заглушки
- краткая инструкция "как запустить/проверить"

## Process (шаги)

1. Plan + список файлов.
2. STOP‑POINT: подтвердить план.
3. Генерация скелета небольшими порциями.
4. Verify: сборка/тест.

## Safety

- запрещаем доменные модели/схемы — только абстракции
- не добавлять зависимости без подтверждения

## Template prompt (копипаст)

> Сгенерируй walking skeleton для …  
> Ограничения: минимальный diff, без новых зависимостей, без доменных моделей/реальной схемы.  
> Сначала план + список файлов + тест‑план. После плана дождись OK.

## Done criteria

- проект запускается
- есть хотя бы 1 тест/проверка
- инструкции понятны

## Pitfalls

- оверинжиниринг вместо walking skeleton
- "готово" без verify
